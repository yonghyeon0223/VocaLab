## 왜 이렇게 했는가

- **verificationCode에 crypto.randomInt 사용**: Math.random은 암호학적으로 안전하지 않다. 인증 코드처럼 보안이 중요한 값은 반드시 Node.js 내장 crypto 모듈을 써야 한다.
- **JWT 서명 검증과 DB 비교 분리**: `verifyRefreshToken`은 서명만 확인한다. DB의 해싱된 값과 비교는 service에서 담당한다. 서명이 유효해도 DB 값과 불일치하면 탈취로 간주해 전체 폐기하는 로직을 service에서 명시적으로 구현할 수 있게 하기 위해서다.
- **mailer 싱글턴 transporter**: 매 발송마다 새 transporter를 만들면 SMTP 연결 비용이 발생한다. 모듈 수준에서 한 번만 생성해 재사용한다.
- **이메일 제목에 코드 포함**: 푸시 알림이나 미리보기에서 앱을 열지 않고도 코드를 확인할 수 있어 UX가 좋아진다.
