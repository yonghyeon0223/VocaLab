# 디자인 시스템

**날짜**: 2026-03-29
**스프린트**: Sprint 01
**상태**: ✅ 완료

---

## 무엇을 했는가

앱 전체에서 재사용할 디자인 시스템을 구축했다. 다크모드 컬러 토큰을 한 곳에 정의하고, 이를 기반으로 Button, TextInput, PasswordInput, Logo 네 개의 공통 컴포넌트를 만들었다. 이제 모든 화면은 이 컴포넌트들을 조합해서 일관된 UI를 구성할 수 있다.

---

## 왜 이렇게 했는가

- **colors.ts 중앙화**: 색상 값을 컴포넌트마다 하드코딩하면 나중에 한 곳만 바꿔도 전체를 수정해야 한다. 토큰 파일 하나만 고치면 앱 전체에 반영되도록 했다.
- **TextInput 상태 관리**: 포커스 상태를 컴포넌트 내부에서 `useState`로 관리한다. 화면 컴포넌트는 에러 메시지만 내려주면 되고, 포커스/블러 시각 처리는 내부에서 알아서 한다.
- **PasswordInput이 TextInput을 래핑**: 중복 코드 없이 포커스/에러 기능을 그대로 쓸 수 있다. 강도 바와 표시/숨김 토글만 추가했다.
- **강도 판단 순서**: 길이 → 영문 → 숫자 → 특수문자 순으로 조건을 쌓는다. 가장 기본적인 조건부터 확인해서 사용자가 어디까지 왔는지 직관적으로 보여준다.
- **Logo size prop**: 인증 화면(large)과 헤더(small) 등 크기가 다른 상황에 대응할 수 있도록 3단계로 분리했다.

---

## 작업 흐름 (Workflow)

1. `colors.ts`를 먼저 작성해 모든 컴포넌트가 동일한 토큰을 참조하게 했다.
2. 가장 단순한 `Button`부터 만들어 기본 패턴을 잡았다.
3. `TextInput`을 만들며 포커스/에러 상태 처리 방식을 정립했다.
4. `TextInput`을 내부에서 쓰는 `PasswordInput`을 만들어 강도 바와 토글을 추가했다.
5. `Logo`로 마무리했다.
