## 트러블슈팅 기록

### 1. PasswordInput 토글 버튼 위치 밀림

**증상**: 비밀번호 확인 필드에서 "비밀번호가 일치하지 않습니다" 에러 메시지가 표시될 때 "표시/숨김" 토글 버튼이 아래로 밀리는 현상.

**원인**: toggle 버튼이 `inputRow`(label + input + error 전체)를 기준으로 `bottom: 14`에 절대 위치했다. error 텍스트가 생기면 `inputRow` 높이가 커지면서 `bottom: 14`가 가리키는 위치도 달라졌다.

**해결**: label과 error를 PasswordInput이 직접 렌더링하고, TextInput에는 넘기지 않도록 변경. toggle 버튼을 입력창(inputBox)에만 겹치는 별도 View 안에 `top: 14`로 고정. label/error는 inputBox 바깥에 있으므로 높이 변화에 영향받지 않는다.

## 추가 고려사항 (다음 스프린트 참고)

### 환경변수
- `app/.env`의 `EXPO_PUBLIC_API_URL`은 **PC의 로컬 IP**로 설정해야 한다. `localhost`는 폰에서 서버에 닿지 않는다.
- `server/.env`의 `JWT_ACCESS_EXPIRES_IN=24h`, `JWT_REFRESH_EXPIRES_IN=365d`로 설정 필요.

### 인증 흐름
- 회원가입 성공 후 로그인 화면으로 이동한다. 사용자가 다시 로그인해야 하는 불편함이 있다. 회원가입 직후 자동 로그인 처리를 Sprint 02에서 개선 검토 가능.
- 현재 `isVerified: false`로 가입되며 이메일 인증은 Sprint 02에서 구현 예정.

### 보안
- 로그인 실패 횟수 제한이 없다. 브루트포스 공격에 노출될 수 있으므로 이후 스프린트에서 rate limiting 추가 필요.
- JWT_SECRET 값이 단순하면 보안에 취약하다. 배포 시 충분히 복잡한 랜덤 문자열로 교체 필요.

### 토큰 관리
- Access Token이 메모리(Zustand)에만 있으므로 앱을 완전히 종료하면 사라진다. `tryAutoLogin`이 Refresh Token으로 재발급해 복원하는 구조다. 재발급이 느리면 스플래시 화면이 길어질 수 있다.
- Refresh Token을 AsyncStorage에 평문 저장한다. 보안이 중요해지면 expo-secure-store로 교체 검토.

### 앱 UX
- 메인 화면은 현재 임시 구성이다. Sprint 03~에서 실제 학습 콘텐츠로 교체된다.
- 키보드가 올라올 때 스크롤 처리를 했지만, 화면이 작은 기기에서는 추가 테스트 필요.
