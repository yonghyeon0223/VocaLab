# 로그인/회원가입 화면 UI 구현

**날짜**: 2026-03-29
**스프린트**: Sprint 01
**상태**: ✅ 완료

---

## 무엇을 했는가

로그인 화면과 회원가입 화면 UI를 구현하고 네비게이터로 연결했다. 디자인 시스템 컴포넌트(Button, TextInput, PasswordInput, Logo)만 조합해 sprint-01.md에 명시된 타이틀/부제목/필드 구성을 그대로 구현했다. 두 화면 사이를 탭으로 이동할 수 있고, 회원가입 화면에는 비밀번호 강도 바가 실시간으로 표시된다.

---

## 왜 이렇게 했는가

- **UI만 먼저 구현**: API 연동과 화면 구성을 한 번에 하면 디버깅이 어렵다. 화면이 의도한 대로 보이는 걸 먼저 확인한 뒤, 다음 작업에서 API를 붙인다.
- **KeyboardAvoidingView + ScrollView**: 키보드가 올라올 때 입력창이 가려지지 않도록 처리했다. iOS와 Android 동작 방식이 달라 behavior를 플랫폼별로 분기했다.
- **에러 상태 관리**: 로그인은 단일 에러 문자열, 회원가입은 필드별 에러 객체로 관리한다. 필드별로 에러를 보여주는 게 사용자 경험상 더 낫기 때문이다.
- **headerShown: false**: 네비게이션 헤더를 숨기고 화면 전체를 직접 제어한다. 인증 화면은 자체 레이아웃이 있어 기본 헤더가 필요 없다.

---

## 작업 흐름 (Workflow)

1. LoginScreen 구현 — 타이틀, 이메일/비밀번호 입력, 로그인 버튼, 회원가입 링크
2. SignupScreen 구현 — 타이틀, 이메일/비밀번호/확인 입력, 강도 바, 계정 만들기 버튼, 이용약관
3. RootNavigator 구현 — Login/Signup 두 화면을 native stack으로 연결
4. App.tsx를 Hello World에서 RootNavigator로 교체
5. TypeScript 컴파일 확인 → 에러 없음
