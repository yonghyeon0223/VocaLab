## 화면 구성

### 홈 화면 (HomeScreen)

**대시보드 레이아웃** (위에서 아래 순서)

```
┌─────────────────────────────────────┐
│  학습 현황 카드 (placeholder)         │
│  - 8단계별 학습 완료 단어 수           │
│  - 오늘 학습 목표 추천                │
│  - 미완료 세트 이동 prompt            │
│  (실제 데이터 연결은 학습 기능 구현 후)  │
├─────────────────────────────────────┤
│  복습 캘린더 (placeholder)            │
│  - 월별 복습 스케줄 시각화             │
│  - 스탬프 시스템                      │
│  (실제 데이터 연결은 장기기억 구현 후)   │
├─────────────────────────────────────┤
│  내 단어 세트 목록                     │
│  - 세트 카드: 이름 / 단어 수 / 생성일   │
```

**룰**
- placeholder 영역은 디자인 틀만 잡고 mock 데이터로 표시
- 단어 세트 목록은 서버에서 실제 데이터를 가져와 표시
- 세트가 없으면 빈 상태 UI 표시 (예: "첫 단어 세트를 만들어보세요!")
- "새 단어 세트 만들기" 버튼 → 세트 생성 플로우 진입

### 단어 세트 생성 플로우

```
[홈 화면 "새 단어 세트 만들기" 버튼]
        ↓
[화면 1] 세트 이름 입력
        ↓
[화면 2] 영단어 일괄 입력
        ↓
서버 저장 → 홈 화면 복귀
```

#### 화면 1 — 세트 이름 입력 (WordSetNameScreen)

**UI**
- 타이틀: "단어 세트 이름을 지어주세요"
- TextInput: 세트 이름 입력
- placeholder: "예: TOEIC 필수 단어"
- 다음 버튼

**룰**
- 최소 1자, 최대 30자
- 공백만으로 구성 불가 (앞뒤 trim)
- 유효하지 않으면 다음 버튼 비활성

---

#### 화면 2 — 영단어 일괄 입력 (WordSetWordsScreen)

**UI**
- 타이틀: "단어를 입력하세요"
- 부제목: "줄바꿈 또는 쉼표로 구분해 주세요. (5~200개)"
- 멀티라인 TextInput: 영단어 일괄 입력 영역
- placeholder: "apple\nbanana\ncherry\n..."
- 단어 수 카운터: "N개 입력됨" (실시간 업데이트)
- 저장 버튼

**룰**
- 줄바꿈(`\n`) 또는 쉼표(`,`)로 단어 구분
- 각 단어는 앞뒤 공백 trim 처리
- 빈 줄 / 빈 항목은 무시
- 중복 단어 자동 제거 (대소문자 무시)
- 최소 5개, 최대 200개
  - 5개 미만: 저장 버튼 비활성 + "최소 5개의 단어가 필요해요"
  - 200개 초과: 저장 버튼 비활성 + "최대 200개까지 입력할 수 있어요"
- 저장 성공 → 홈 화면으로 복귀 (세트 목록에 새 세트 표시)

## DB 모델

### `wordSets` 컬렉션

```typescript
type WordSet = {
  _id: ObjectId;
  userId: ObjectId;        // users 컬렉션 참조
  name: string;            // 세트 이름 (1~30자)
  wordCount: number;       // 세트 내 단어 수 (비정규화)
  createdAt: Date;
  updatedAt: Date;
}
```

인덱스: `{ userId: 1, createdAt: -1 }` (유저별 최신순 조회)

### `words` 컬렉션

```typescript
type Word = {
  _id: ObjectId;
  wordSetId: ObjectId;     // wordSets 컬렉션 참조
  userId: ObjectId;        // users 컬렉션 참조
  spelling: string;        // 영단어 원형 (소문자 정규화)
  createdAt: Date;
}
```

인덱스: `{ wordSetId: 1 }` (세트별 단어 조회)

> **참고**: 뜻, 예문, 파생어, 음성 등 학습 콘텐츠 필드는 다음 스프린트에서 데이터 파이프라인 구현 시 추가한다. Sprint 04에서는 영단어 원형만 저장한다.

## API 엔드포인트

### POST /api/word-sets
```
미들웨어: authenticate
요청: { name: string, words: string[] }
처리:
  1. name 검증 (1~30자, 공백만 불가)
  2. words 검증 (5~200개, 중복 제거, trim, 소문자 정규화)
  3. wordSets 컬렉션에 세트 생성
  4. words 컬렉션에 단어 일괄 삽입
성공: 201 { success: true, data: { wordSet } }
실패: 400 name 유효성 위반
      400 words 개수 범위 위반 (5~200)
```

### GET /api/word-sets
```
미들웨어: authenticate
요청: 없음
처리: 해당 유저의 단어 세트 목록 조회 (최신순)
성공: 200 { success: true, data: { wordSets: WordSet[] } }
```

### GET /api/word-sets/:id
```
미들웨어: authenticate
요청: 없음
처리: 단어 세트 상세 조회 (세트 정보 + 단어 목록)
성공: 200 { success: true, data: { wordSet, words: Word[] } }
실패: 404 세트 없음 또는 권한 없음
```

### DELETE /api/word-sets/:id
```
미들웨어: authenticate
요청: 없음
처리:
  1. 세트 소유권 확인
  2. words 컬렉션에서 해당 세트 단어 전부 삭제
  3. wordSets 컬렉션에서 세트 삭제
성공: 200 { success: true }
실패: 404 세트 없음 또는 권한 없음
```
