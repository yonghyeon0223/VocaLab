# 프로젝트 초기 세팅 (Sprint 00)

**날짜**: 2026-03-29
**스프린트**: Sprint 00
**상태**: ✅ 완료

---

## 무엇을 했는가

Express 서버와 Expo 앱의 실행 가능한 최소 환경을 구성했다. 서버는 환경변수를 로딩하고 MongoDB에 연결한 뒤 지정된 포트에서 요청을 받을 수 있는 상태가 됐고, 앱은 Expo Go에서 "Hello World"를 화면에 표시할 수 있는 상태가 됐다. 이 세팅 위에 Sprint 01부터 실제 기능을 쌓아 올릴 수 있다.

---

## 왜 이렇게 했는가

- **MongoDB native driver 선택**: `mongoose` 사용이 금지되어 있어 native driver를 직접 사용. `connectDB()` 함수로 연결을 모듈 수준에서 관리해 매 요청마다 재연결하는 낭비를 막았다.
- **환경변수 중앙 관리**: `server/src/utils/env.ts`에서 모든 환경변수를 한 곳에 모았다. 값이 없을 때 빈 문자열 대신 명확한 기본값을 두어 서버 실행 시 어떤 값이 빠졌는지 바로 알 수 있다.
- **DB 연결 후 서버 오픈**: `bootstrap()` 함수에서 `connectDB()` 완료 후 `app.listen()`을 호출한다. DB 없이 요청을 받는 상황을 원천 차단하기 위해 순서를 강제했다.
- **shared/types.ts 뼈대만 작성**: 세부 타입은 각 스프린트에서 필요할 때 추가한다. 지금은 도메인 용어 정의만 담아 오버엔지니어링을 피했다.

---

## 작업 흐름 (Workflow)

1. 폴더 구조를 먼저 잡아 앞으로 만들 파일들이 어디 들어갈지 뼈대를 세웠다.
2. TypeScript 설정으로 server와 app이 각자 독립적으로 컴파일될 수 있게 했다.
3. 서버 진입점(`index.ts`)을 만들어 환경변수 로딩 → DB 연결 → 서버 실행 흐름을 잡았다.
4. `shared/types.ts`에 도메인 용어를 타입으로 선언해 클라이언트와 서버가 같은 언어로 소통할 기반을 만들었다.
5. Expo 앱을 초기화하고 Hello World 화면을 출력하는 최소 구성을 완성했다.
6. `.env.example`과 `.gitignore`로 민감 정보가 git에 올라가지 않도록 했다.
7. GitHub에 초기 커밋을 푸시해 협업과 버전 관리 기반을 완성했다.
